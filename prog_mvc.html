Introducción

El patrón Modelo-Vista-Controlador (MVC) es una arquitectura de software que separa la lógica de la aplicación en tres componentes principales:

  • Modelo (Model): Representa los datos y la lógica de negocio.
  • Vista (View): Muestra la información al usuario.
  • Controlador (Controller): Maneja la interacción entre el modelo y la vista.

Ventajas

  • Separación de responsabilidades: Cada componente tiene una función específica, facilitando la organización del código.
  • Facilidad de mantenimiento: Separar la lógica de negocio, la presentación y la gestión de la interacción permite modificar o mejorar cada parte sin afectar a las demás.
  • Reutilización de código: La vista y el modelo pueden reutilizarse en diferentes partes del programa o incluso en otros proyectos.

Ventajas

  • Escalabilidad: Al estar bien estructurado, es más fácil añadir nuevas funcionalidades sin afectar el sistema en su conjunto.
  • Facilita la colaboración: Diferentes desarrolladores pueden trabajar simultáneamente en distintas partes del código sin interferencias.
  • Mejor prueba y depuración: Permite realizar pruebas unitarias más eficaces, ya que la lógica de negocio está separada de la interfaz gráfica.

Ejemplo básico con una clase base

1. Modelo: Define los datos y la lógica de negocio.

						
							class Estudiante {
								private String nombre;
								private int edad;
								
								public Estudiante(String nombre, int edad) {

									this.edad = edad;
								}

								}
								
								public void setNombre(String nombre) {
									this.nombre = nombre;
								}
								
								public int getEdad() {
									return edad;
								}
								
								public void setEdad(int edad) {
									this.edad = edad;
								}
							}

Ejemplo básico con una clase base

2. Vista: Muestra información al usuario.

						
							class EstudianteVista {
								public void mostrarEstudiante(String nombre, int edad) {
									System.out.println("Estudiante: ");
									System.out.println("Nombre: " + nombre);
									System.out.println("Edad: " + edad);
								}
							}

Ejemplo básico con una clase base

3. Controlador: Gestiona la relación entre el modelo y vista.

						
							class EstudianteControlador {
								private Estudiante modelo;
								private EstudianteVista vista;
								
								public EstudianteControlador(Estudiante modelo, EstudianteVista vista) {
									this.modelo = modelo;
									this.vista = vista;
								}
								
								public void setNombreEstudiante(String nombre) {
									modelo.setNombre(nombre);
								}
								
								public String getNombreEstudiante() {
									return modelo.getNombre();
								}
								
								public void setEdadEstudiante(int edad) {
									modelo.setEdad(edad);
								}
								
								public int getEdadEstudiante() {
									return modelo.getEdad();
								}
								
								public void actualizarVista() {
									vista.mostrarEstudiante(modelo.getNombre(), modelo.getEdad());
								}
							}

Ejemplo básico con una clase base

4. Programa principal: Conectamos los componentes y probamos el patrón MVC.

						
							public class MVCDemo {
								public static void main(String[] args) {
									// Crear modelo
									Estudiante modelo = new Estudiante("Juan Perez", 20);
									
									// Crear vista
									EstudianteVista vista = new EstudianteVista();
									
									// Crear controlador
									EstudianteControlador controlador = new EstudianteControlador(modelo, vista);
									
									// Actualizar vista inicial
									controlador.actualizarVista();
									
									// Modificar el modelo a través del controlador
									controlador.setNombreEstudiante("Ana Gómez");
									controlador.setEdadEstudiante(22);
									
									// Actualizar vista con nuevos datos
									controlador.actualizarVista();
								}
							}

Ejemplo completo con varias clases base

Vamos a implementar un ejemplo de MVC donde gestionamos la información de un estudiante que puede cursar muchas asignaturas.

Ejemplo completo con varias clases base

1. Modelo: Define los datos y la lógica de negocio.

						
							// Archivo Asignatura.java
							class Asignatura {
								private String nombre;
								private int creditos;
								
								public Asignatura(String nombre, int creditos) {
									this.nombre = nombre;
									this.creditos = creditos;
								}
								
								public String getNombre() {
									return nombre;
								}
								
								public int getCreditos() {
									return creditos;
								}
							}
							
							// Archivo Estudiante.java
							class Estudiante {
								private String nombre;
								private int edad;
								private Asignatura[] asignaturas;
								private int numAsignaturas;
								
								public Estudiante(String nombre, int edad, int maxAsignaturas) {
									this.nombre = nombre;
									this.edad = edad;
									this.asignaturas = new Asignatura[maxAsignaturas];
									this.numAsignaturas = 0;
								}
								
								public String getNombre() {
									return nombre;
								}
								
								public void setNombre(String nombre) {
									this.nombre = nombre;
								}
								
								public int getEdad() {
									return edad;
								}
								
								public void setEdad(int edad) {
									this.edad = edad;
								}
								
								public void agregarAsignatura(Asignatura asignatura) {
									if (numAsignaturas < asignaturas.length) {
										asignaturas[numAsignaturas++] = asignatura;
									}
								}
								
								public Asignatura[] getAsignaturas() {
									return asignaturas;
								}
								
								public int getNumAsignaturas() {
									return numAsignaturas;
								}
							}

Ejemplo completo con varias clases base

2. Vista: Muestra información al usuario.

						
							class EstudianteVista {
								public void mostrarEstudiante(String nombre, int edad, Asignatura[] asignaturas, int numAsignaturas) {
									System.out.println("Estudiante: ");
									System.out.println("Nombre: " + nombre);
									System.out.println("Edad: " + edad);
									System.out.println("Asignaturas: ");
									for (int i = 0; i < numAsignaturas; i++) {
										System.out.println("- " + asignaturas[i].getNombre() + " (" + asignaturas[i].getCreditos() + " créditos)");
									}
								}
							}

Ejemplo completo con varias clases base

3. Controlador: Gestiona la relación entre el modelo y vista.

						
							class EstudianteControlador {
								private Estudiante modelo;
								private EstudianteVista vista;
								
								public EstudianteControlador(Estudiante modelo, EstudianteVista vista) {
									this.modelo = modelo;
									this.vista = vista;
								}
								
								public void setNombreEstudiante(String nombre) {
									modelo.setNombre(nombre);
								}
								
								public String getNombreEstudiante() {
									return modelo.getNombre();
								}
								
								public void setEdadEstudiante(int edad) {
									modelo.setEdad(edad);
								}
								
								public int getEdadEstudiante() {
									return modelo.getEdad();
								}
								
								public void agregarAsignaturaEstudiante(Asignatura asignatura) {
									modelo.agregarAsignatura(asignatura);
								}
								
								public void actualizarVista() {
									vista.mostrarEstudiante(modelo.getNombre(), modelo.getEdad(), modelo.getAsignaturas(), modelo.getNumAsignaturas());
								}
							}

Ejemplo completo con varias clases base

4. Programa principal: Conectamos los componentes y probamos el patrón MVC.

						
							public class MVCDemo {
								public static void main(String[] args) {
									// Crear modelo
									Estudiante modelo = new Estudiante("Juan Perez", 20, 5);
									
									// Crear vista
									EstudianteVista vista = new EstudianteVista();
									
									// Crear controlador
									EstudianteControlador controlador = new EstudianteControlador(modelo, vista);
									
									// Agregar asignaturas
									controlador.agregarAsignaturaEstudiante(new Asignatura("Matemáticas", 6));
									controlador.agregarAsignaturaEstudiante(new Asignatura("Historia", 4));
									
									// Actualizar vista inicial
									controlador.actualizarVista();
									
									// Modificar el modelo a través del controlador
									controlador.setNombreEstudiante("Ana Gómez");
									controlador.setEdadEstudiante(22);
									controlador.agregarAsignaturaEstudiante(new Asignatura("Programación", 8));
									
									// Actualizar vista con nuevos datos
									controlador.actualizarVista();
								}
							}

Resumen:

  • Se crea un objeto Estudiante con un array de asignaturas.
  • Se define Asignatura para representar cada materia con nombre y créditos.
  • La clase EstudianteVista muestra los datos en la consola, incluyendo las asignaturas.
  • El EstudianteControlador permite modificar el modelo y actualizar la vista.
  • En MVCDemo, se instancian los componentes y se manipulan los datos a través del controlador.
